游戏设计与逻辑

首先开机后会进入一个开机画面，点击OK确认后，进入菜单选择，分别是键控和声控两个选项，根据选择显示键控（或声控）被选择的画面，确定选择之后点击OK即可正式开始游戏。

游戏的状态控制：初始化的时候记录菜单选择，游戏进行过程中根据玩家的选择控制中断中的行为，每一帧进行人物位置判断的时候，会结合地图位置来进行游戏状态判断，当人物达到死亡条件时，游戏不再进行，蜂鸣器报警并显示分数画面，玩家可选择重新开始。

游戏的计分系统：记录玩家存活时间作为游戏的分数，在本逻辑中每过一帧计一分。

游戏的初始地图是一个固定高度，宽度有限制随机的矩形数组集合，人物的位置也是随机（算法确保人物初始落在地面上，且距离“悬崖”较远），然后通过玩家在键控模式的按键中断，或者声控模式达到一定阈值的中断，结合人物的位置，来判断是否进行跳跃。其中，人物移动的每一帧都会结合地图及是否有中断信息来判断人物的位置、游戏的状态、同时使地图向前刷新。简化后的游戏状态模型可以参见游戏状态图。

Elastic Search 学习时看到的story

Posted on 2018-05-13 | In Magic

许多年前，一个刚结婚的名叫 Shay Banon 的失业开发者，跟着他的妻子去了伦敦，他的妻子在那里学习厨师。 在寻找一个赚钱的工作的时候，为了给他的妻子做一个食谱搜索引擎，他开始使用 Lucene 的一个早期版本。

直接使用 Lucene 是很难的，因此 Shay 开始做一个抽象层，Java 开发者使用它可以很简单的给他们的程序添加搜索功能。 他发布了他的第一个开源项目 Compass。

第一个公开版本在2010年2月发布，从此以后，Elasticsearch 已经成为了 Github 上最活跃的项目之一，他拥有超过300名 contributors(目前736名 contributors )。 一家公司已经开始围绕 Elasticsearch 提供商业服务，并开发新的特性，但是，Elasticsearch 将永远开源并对所有人可用。

函数传参到底是值传递

Posted on 2018-05-03 | In keep-learning

结论：

红宝书上说都是值传递

传递对象的时候也是值传递，本来传递的东西就是该对象的引用，所以全是值传递！

内部机制：

其实函数接受参数之前，会先在函数作用域内创建一个局部变量，并把传入的参数赋予创建的局部变量，如果值类型传递，就是重新开辟一块内存存储传入变量的值，并把新创建的变量指向此内存地址，因此对此变量的修改并不会影响到外部变量的值；

如果传入的不是值类型，则仅仅是对变量进行引用赋值操作，只是把新创建的局部变量指向外部变量的地址，因此对局部变量的修改就相当于对外部变量的修改。

JS 内存泄露

Posted on 2018-04-25 | In keep-learning

Memory leak patterns in JavaScript

paper in 2007

IE和Mozilla Firefox：在JS中最容易内存泄露的浏览器。问题的根源是组件对象模型拿来管理dom对象，Windows的COM 组件对象模型，和Mozilla的XPCOM（跨平台组件对象模型）关于内存的分配和失而复得都用的reference-counting型垃圾收集，而reference-counting 机制并不能与JS的mark-and-sweep机制一直友好相处

JS是一个垃圾收集语言，意味着内存被释放，在对象被浏览器创建，且已经没有被引用的时候。虽然JS的垃圾回收机制没有任何毛病，但它与一些浏览器处理DOM对象的内存释放与回收不太一致。

IE和Firefox在这个问题上使用的是reference counting 的方式。 在一个reference counting 系统中，每个有引用的object 维持一个“有多少对象正在引用它”的计数，如果计数为0，这个对象将会被销毁并且内存回到堆顶。虽然这个解决方案通常非常有效，但是它有一个盲点，当它被循环引用的时候。

循环引用有什么问题呢？

循环引用是有两个互相指向对方的对象组成的，他们各给对方一个为1的计数。在一个纯粹的垃圾回收机制中，循环引用是没有问题的：如果两个被需要的对象都没有被其他任何对象引用，那么它们会被回收。在一个reference counting系统中，他们都不会被回收，因为计数永远不会达到为0。在一个混用系统中，当这两种机制一起被使用的时候，会因为系统无法区分一个循环引用而导致内存泄露，DOM对象或者JS对象都不会被销毁。

解决方案

标记-清除算法

引用计数法是将垃圾定义为没有被引用的内存，而标记-清除算法是将其定义为对象是否可以被获得。

定期从root开始向下寻找所有对象，垃圾回收记住所有的可被找到和不被找到的对象，并且将不被找到的对象回收。

从12年开始，所有现代浏览器都使用了标记-清除算法。

计算机系统基础 读书笔记 第一章

Posted on 2018-04-17 | In note

1 基础

QUESTION: HOW TO BUILD A NETWORK

计算机网络的主要特征是通用性。

得到网页的过程模型：

  • 域名解析(应用了UDP) => 建立TCP连接 => 浏览器发出请求，服务器返回页面 => 关闭TCP

构建计算机网络需要：

  • 确定影响网络设计的所有需求和约束。

    • 应用程序员：所需服务
    • 网络设计者： 高效
    • 网络提供者：易于执行管理

  • 连通性

    • 网络组件：节点、链路
    • 方式（从易到难）
      • 直连（一条链路，一条总线）
      • 间接连结（更多的链路，交换网络（常见类型：电路交换和分组交换），交换机（存储和转发分组））
      • 网络互联（多个网络，路由器/网关）
    • 网络互联关键
      • 为网络中每一个可到达的节点定一个地址，从而利用这个进行路由选择然后正确传送消息。

  • 有效性

    • 多路复用：多用户共享系统资源

      • 同步时分多路复用：STDM

      • 频分多路复用： FDM

      • 统计多路复用（按需而不是时间片的STDM）

        STDM：相当于传送带的固定位置占了固定的人，当接收数据的盒子经过一个人A之后，盒中数据被取出并到达下一个地方B，B装载数据，等待一个时间片的时间，再取出数据使下一个人C装载数据。其中ABC分别装载数据的时间是固定的一个时间片。给每个主机分配一段时间。

        FDM ：在传输数据的盒子中给每个主机分配一个位置，每个位置代表一个频率。

        以上两种有 时间片空闲或者位置空闲的情况，浪费资源。

        统计多路复用： 按照时间共享物理链路，传送带只有一个位置，需要的人就上前使用这个位置，并在规定的时间片内装载数据，接着数据传送给交换机，然后换下来给另外有需要的人用。

        其中，为了保证所有流都有机会传送数据，每个流在指定时间内只能传送一定大小的数据块（分组 ），超出则在这段时间都没有机会上前了。因此还需要主机（源节点）将消息化成几个分组，接收者重组还原消息。

        位置的使用权：以分组接分组为基础决定下一个发送分组的流。也相当于轮流发。

        这种方法可以避免空闲时间，使分组交换具有高效。
